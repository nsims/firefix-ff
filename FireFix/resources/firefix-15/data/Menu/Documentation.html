Request Creator Documentation

FireFix

Introduction

FireFix is a Mozilla FireFox pluggin which allows to post requests on Software in a minimum of time and increases the quality of requests by filling often forgotten information.

In concret terms, when one finds a bug or wants to ask for an evolution, one just need to go on the relevant tab and click on FireFix's icon. At this moment FireFix:

  • Takes a screenshot of the whole tab
  • Retrieves the error message if one is displayed
  • Gathers all the technical information needed to fill the request (product, application server, builds, url,..)
  • Opens a new tab containing the prefilled request
In about 2 seconds the boring work is done, one just have to fill in the user story and the context then post the request.

The main features of Firefix will be described in the following paragraphs along with FireFix limitations and coming evolutions.

Main Menus

FireFix has 2 menus. The first one allows the user to define default values that will be used in priority to prefill the requests. This becomes especially useful when taking part in a X-Test.

The second menu contains the Software's login and password, it is used by FireFix when creating a request. If the password is out-of-date, Firefix detects it and asks for an update.

Icon Status

FireFix has different icons representing its current state. It depends on the web page currently visited and, if the web page is an Enablon application, the settings of the application.

  •   Enablon Application, FireFix is working.
  •   Not an Enablon Application or on Software, FireFix is deactivated.
  •   Enablon Application BUT User not Logged, FireFix is deactivated.
  •   Enablon Application BUT Version Page not accessible or doesn't exist, FireFix is deactivated.

Product Selection

In order to detect the product, FireFix uses the Package/Module/Application's label (depends on the delivery mode) and compares it to the ID of the application. This works most of the times, however it happens that the labels don't match (Ex, in the case of the Action Plans : AP != AP_Pkg).

In this case FireFix displays a popup listing the names of the differents Package/Module found and the user has to choose the one he thinks the most relevant.

Application Server Selection

When prefilling a request, FireFix check that the application server used in the application and the application server linked to the product found on Software are the same. If this is not the case it can have 3 causes :

  • Software is not up to date (please contact the team responsible of the product so that they can make the necessary changes).
  • FireFix has found the wrong product (please send a mail to one of the members of the FireFix development team so that we can find a solution to the issue).
  • The product has been moved to a more recent application server.

If this happens Firefix displays a popup explaining the problem and ask the user to choose the socle to be used in the request.

Limitations

In order to work properly FireFix needs to be executed in a specific context. This leads to some limitations.

  • The Version Page has to be accessible to the user(Path: siteID/go.asp?u=/AppID/ver). This is where the product information is found.
  • FireFix is fully compatible with the following versions of the application server : 5.7SP1, 6.0-R6, 6.0-R7, Alderaan, and more recent. In the older versions, FireFix is not able to find the product no matter the application settings.
  • When working on the serveur dev2 with a local account, using FireFix is possible but will disconect the user from the application. This can't be avoided as FireFix needs to connect to Software in order to create the request.

What Next?

Here is the list of the coming evolutions to be added to FireFix. It is not set in stone so feel free to contact the developer team if you have any query or evolution idea.

  • Add more options to the Screenshot (for exemple allowing to customize it).
  • Add the notion of Project in FireFix (new field on the Software's request table).
  • Create a X-Test Mode, the user tick a box and FireFix prefill the requests in a standardized way which has to be defined.
  • Application Status Management, when connecting to an application, FireFix check the build of the product and warn the user if it is not up-to-date.
